# AI牛马刷题 — 端侧推理比赛演示脚本

本文档为比赛评委和演示人员提供一份清晰的端侧推理（MNN + Qwen）演示操作指南。

## 前置条件

开始演示前请确认以下条件已满足：

1. **Android 设备**：搭载 ARMv8.2+ 处理器的 Android 手机或平板（推荐骁龙 8 Gen 1 及以上）
2. **已编译 APK**：项目已通过 Android Studio 构建，APK 已安装到设备
3. **模型已加载**：进入个人页后，MNN 状态区显示"模型已就绪"，未显示错误提示。如显示"待加载"，请先点击"加载本地模型"按钮完成加载（首次加载约需 30-60 秒，后续启动自动加载）
4. **可选 — 断网验证**：演示前关闭 Wi-Fi 和移动数据，向评委证明推理过程 100% 离线运行

## 演示步骤

### 第一步：确认模型状态（个人页）

1. 打开应用，点击底部导航栏"我的"，进入个人页
2. 向下滚动至"MNN 端侧模型"区域
3. 确认显示"模型已就绪"或类似状态提示
4. 如果显示异常，参考故障排查部分

### 第二步：使用演示材料出题

1. 在个人页"MNN 端侧模型"区域内，找到"演示材料"按钮
2. 点击"使用演示材料体验端侧出题"
3. 应用自动跳转到刷题首页，输入框已填入一段急救知识材料（约 300 字）
4. 确认出题模式为"原文提取"，题型为"单选题"
5. 确认题量为 3-5 题（默认 5 题）
6. 点击"开始出题"

### 第三步：观察出题过程

1. 页面显示"AI 出题中..."加载提示
2. 由于端侧推理在本地 CPU 完成，出题过程约需 15-45 秒（取决于设备性能和题量）
3. 出题完成后自动跳转到做题界面，展示生成的题目列表

### 第四步：验证题目质量

1. 查看生成的题目，确认：
   - 每道题有清晰的题干和 4 个选项
   - 题目内容与输入的急救知识材料相关
   - 选项中有明确的一个正确答案
2. 作答 1-2 道题，验证即时反馈功能正常
3. 返回题集页，确认题目已沉淀到"全部"分类中

### 第五步（可选）：演示自定义材料出题

1. 返回刷题首页，手动粘贴一段学习材料（建议 800-1500 字）
2. 调整题量和难度
3. 再次出题，展示端侧推理的通用性

## 预期效果

- 每轮出题耗时约 15-45 秒（取决于设备 CPU 性能和题量）
- 生成的题目与输入材料高度相关，格式规范
- 全程无网络请求（可在飞行模式下验证）
- 题目自动进入题集，支持错题和掌握状态管理

## 故障排查

| 现象 | 可能原因 | 解决方案 |
|------|---------|---------|
| 个人页不显示 MNN 区域 | 未设置 `VITE_LLM_RUNTIME=local-mnn` | 检查 `.env.local` 或环境变量配置 |
| "模型加载失败" | 模型文件缺失或路径错误 | 确认 `.mnn` 文件已放入对应 libs 目录 |
| 出题超时 | 材料过长或设备性能不足 | 减少题量至 3 题，缩短材料至 500 字以内 |
| 题目格式异常 | 模型输出 JSON 解析失败 | 重试出题，系统内置 JSON 格式修复机制 |
| 应用闪退 | MNN so 未正确链接或 ABI 不匹配 | 确认 CMake 编译通过，MNN so 存在于 jniLibs |

## 演示要点（给评委的 highlights）

在演示过程中可以强调以下技术亮点：

1. **100% 离线推理**：关闭网络后仍可正常出题，数据完全不出设备
2. **ARM 指令级优化**：MNN 利用 ARMv8.2+ 和 SME2 指令集加速，在移动端 CPU 上实现可用的推理速度
3. **端到端闭环**：从材料输入 → 本地推理出题 → 做题反馈 → 题集沉淀，完整学习闭环在端侧跑通
4. **智能降级**：系统自动感知端侧模式，调整 prompt 长度、题量上限等参数，确保端侧模型也能产出高质量题目
5. **JSON 格式修复**：内置格式修复重试机制，提升端侧模型 JSON 输出的可靠性
